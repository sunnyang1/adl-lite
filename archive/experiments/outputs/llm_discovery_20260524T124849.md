# Peripheral Attention Trap

## Discovery Statement

In AML transaction monitoring, alert-generation logic that weights central graph nodes (high-degree hubs, known typology anchors) creates a systematic blind spot for illicit fund flows routed through low-degree peripheral nodes. When multiple peripheral nodes converge into a shared sink entity, the aggregate risk signal exceeds any individual node's threshold, yet the monitoring system fails to flag the pattern because each peripheral edge remains below the attention cutoff. The emergent trap arises not from a single evasion technique but from the structural interaction between graph topology and threshold-based attention allocation.

## Intuition

Standard AML graph analytics assign attention proportionally to node centrality. Peripheral nodes — entities with few connections, short transaction histories, or low cumulative volumes — receive minimal scrutiny. Sophisticated launderers exploit the peripheral zone by distributing activity across dozens of low-degree nodes that individually appear benign. The critical insight is that **sink convergence** — where multiple peripheral nodes funnel funds into a single destination — creates a high-risk subgraph that remains invisible under node-level attention scoring. The trap is emergent because no single peripheral node triggers an alert, yet the collective topology exhibits clear laundering signatures: fan-in geometry, temporal clustering, and amount homogeneity across peripheral edges.
