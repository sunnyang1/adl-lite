# Peripheral Attention Trap

## Discovery Statement

In AML transaction monitoring, alert triage systems and human analysts allocate disproportionate cognitive and computational attention to high-degree hub nodes within a transaction graph, while structurally significant peripheral nodes—nodes with low degree but high betweenness centrality or membership in sink convergence paths—escape scrutiny. The resulting blind zone allows launderers to route funds through sparse, low-visibility subgraphs that converge into a final sink account, exploiting the inherent attention gradient between graph center and periphery.

## Intuition

The "Peripheral Attention Trap" emerges from a compositional blend of two well-documented phenomena: the hub-bias in network anomaly detection (where scoring functions weight node degree heavily) and the sink convergence pattern in layering schemes (where multiple low-degree feeder accounts funnel into a single extraction point). When a monitoring system's attention budget is finite, the system naturally starves peripheral nodes of review cycles. Laundering architects exploit the starvation window by distributing initial placement across many peripheral nodes, each individually below alert thresholds, while the aggregate flow converges into a sink node that also appears benign because each individual incoming edge is small. The trap is emergent: no single peripheral node triggers an alert, yet the collective peripheral-to-sink flow constitutes the laundering payload.
