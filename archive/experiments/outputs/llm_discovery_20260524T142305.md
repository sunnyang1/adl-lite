# Peripheral Attention Trap

## Discovery Statement

In transaction monitoring systems, alert triage workflows exhibit a systematic bias toward graph-peripheral nodes—entities with low degree centrality but high transactional velocity. Analysts allocate disproportionate review cycles to peripheral nodes because sink-convergence heuristics flag unusual inflows without contextualizing the broader network topology. The result is a resource trap: peripheral nodes absorb monitoring attention while high-centrality hub nodes continue operating with reduced scrutiny, allowing coordinated layering to persist undetected across the network core.

## Intuition

The Peripheral Attention Trap emerges from the interaction between two well-known AML monitoring signals: graph peripheral nodes and sink convergence patterns. When a sink-convergence rule fires on a peripheral entity receiving aggregated funds from multiple sources, the alert volume appears high relative to the entity's historical baseline. Triage queues prioritize alerts matching known mule-account typologies. However, the peripheral node often serves as a decoy—a structurally expendable endpoint designed to absorb investigative resources. The true layering operation proceeds through hub nodes whose centrality makes individual transactions appear routine. The trap is emergent: no single rule creates the bias, but the combined effect of sink-convergence prioritization and peripheral-node alerting produces a systematic blind spot at the network core.
