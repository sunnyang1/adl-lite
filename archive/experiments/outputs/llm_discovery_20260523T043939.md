# Peripheral Attention Trap

## Discovery Statement

In AML transaction monitoring, a recurring failure mode emerges when compliance systems concentrate investigative resources on high-centrality nodes within a transaction graph while structurally significant activity migrates to peripheral nodes that feed into a small number of sink accounts. The peripheral nodes individually appear benign—low volume, no direct links to sanctioned entities, no velocity anomalies—but collectively form a convergence pattern that delivers laundered funds to the same destination accounts flagged by the central cluster. Investigative attention on the central cluster creates a blind spot: the peripheral convergence pathway operates below per-node alerting thresholds yet achieves the same fund-movement objective. Detection requires monitoring graph-level sink convergence metrics rather than node-level risk scores alone.

## Intuition

Standard AML rule engines evaluate each node against threshold-based triggers: transaction count, cumulative amount, counterparty risk score. A peripheral node with three small transfers to three different counterparties over two months triggers no alert. However, when two hundred such peripheral nodes all share a common sink account as a final destination, the aggregate flow represents a significant laundering channel. The trap arises because human analysts and automated triage systems allocate attention proportionally to node-level alert severity, leaving the peripheral convergence pattern unexamined until a retrospective investigation reveals the sink account's true inflow structure.

The concept aligns with the attention-trap framework described in `data/aml/concepts/aml-attention-trap.md`, specifically the graph-peripheral-nodes and sink-convergence signal dimensions.
