# Peripheral Attention Trap

## Discovery Statement

In transaction monitoring systems, illicit actors exploit the periphery of the attention graph—nodes with low centrality scores and minimal edge weight—to stage fund movements. The monitoring engine allocates computational attention proportional to node centrality, creating a structural blind spot at the graph periphery. Funds staged through peripheral nodes converge into a sink node only after the attention window closes, producing a delayed detection pattern where the sink node appears benign during the active monitoring phase.

## Intuition

The attention allocation mechanism in AML monitoring mirrors the human cognitive bias toward central, high-activity nodes. Peripheral nodes—nodes with few connections, low transaction volume, or distant graph-theoretic distance from known high-risk clusters—receive disproportionately low monitoring weight. Adversaries construct transaction chains that remain entirely within the peripheral subgraph until a final convergence step. The convergence event triggers alerts only after the peripheral staging phase completes, at which point the sink node has already absorbed funds from multiple peripheral sources. The trap emerges from the interaction between attention-weighted monitoring and adversarial graph traversal.
