# Peripheral Attention Trap

## Discovery Statement

In AML transaction monitoring, a recurring failure mode emerges when compliance systems concentrate investigative attention on high-velocity hub accounts while structurally equivalent peripheral nodes accumulate suspicious activity below alert thresholds. The peripheral attention trap describes the phenomenon where launderers deliberately route funds through low-centrality graph positions—nodes with few direct connections but critical bridging roles—to exploit the monitoring system's implicit bias toward hub-centric risk scoring. Over time, peripheral nodes converge into sink clusters that drain illicit value undetected, creating a blind spot that scales with graph size.

## Intuition

Standard AML graph analytics assign risk scores proportional to node degree, transaction volume, and centrality metrics. Peripheral nodes—nodes with degree below the median but positioned on shortest paths between clusters—receive disproportionately low attention scores. Laundering networks exploit structural isomorphism between a legitimate low-activity account and a deliberately placed mule account. The trap activates when multiple peripheral nodes begin exhibiting coordinated sink convergence: funds flow inward from diverse sources but never re-emerge, forming a terminal absorption pattern. Detection requires monitoring the second-order neighborhood of peripheral nodes rather than the nodes themselves.
