# ADL Lite Coq Formalisation

Machine-checked proofs for the ADL Lite event-chain invariants: status lattice (δ), confidence aggregation (γ), well-formedness (WF), and CRDT merge (T9). All 12 well-formedness axioms are fully defined (no stubs). Iris separation-logic files model the split-lock concurrent design.

## Quick Start

```bash
cd formal/coq
make            # compile all .v files (~2–4 min on a modern laptop)
make validate   # run coqchk (~1–2 min)
make html       # generate Coqdoc HTML
make clean      # remove all generated objects
```

## Key Definitions

### δ — Derived Status (`Chain.v`, line 103)

```coq
Definition derived_status (es : chain) : status :=
  status_lub (map StatusOf (map event_type es)).
```

The derived status of a chain is the **Least Upper Bound (LUB)** over the lifecycle statuses of all events in the chain. `StatusOf` maps each event type to its status (e.g., `VALIDATE → VALIDATED`, `REGISTER → PROVISIONAL`). `status_lub` computes the maximum via `fold_left status_max`, giving the lattice join.

### γ — Derived Confidence (`Chain.v`, line 107)

```coq
Definition derived_confidence (es : chain) : nat :=
  derived_confidence_events es.
```

Where `derived_confidence_events` is the G-Counter max over `VALIDATE` and `SNAPSHOT` events:

```coq
Definition derived_confidence_events (es : list event) : nat :=
  max_confidence (filter (fun e => match event_type e with
                                     | VALIDATE => true
                                     | SNAPSHOT => true
                                     | _        => false
                                     end) es).
```

Confidence is a **G-Counter** with `max` as the join and `0` as the bottom. Lower confidence cannot downgrade an existing higher value.

### WF — Well-Formedness (`Chain.v`, line 135)

```coq
Definition well_formed (es : chain) : Prop :=
  (forall e, In e es -> axiom_valid_event e)
  /
  ... (* 12 axioms *)
```

A well-formed chain satisfies **12 axioms** simultaneously:

1. **Valid event** — non-empty actor string
2. **Increasing IDs** — strictly monotonic `event_id`
3. **Prev linkage** — cryptographic chain (`prev e = Some (event_id predecessor)`)
4. **Scope ACL** — non-empty scope for every actor
5. **Precondition evaluation** — lifecycle events obey `allowed_transition`
6. **Signature verification** — signed events have valid signatures
7. **SHACL constraints** — structural data-shape checks (`event_id > 0`, non-empty actor)
8. **Status transition** — exactly one `REGISTER` at genesis
9. **Confidence clamped** — `confidence e <= MAX_SCALED` (100)
10. **Lifecycle monotonic** — derived status never regresses
11. **Validator collusion** — `VALIDATED` status requires at least one `VALIDATE` event
12. **Synthetic tagging** — events with signatures have public keys

### CRDT Merge (`CRDT.v`, line 423)

```coq
Definition merge_branch (b1 b2 : branch) : branch :=
  reanchor (sort_by_id (dedup (b1 ++ b2))).
```

Merge follows a **sort-dedup-reanchor** pipeline:
1. **Concatenate** the two branches (`b1 ++ b2`)
2. **Deduplicate** by `event_id` (`dedup`)
3. **Sort** by `event_id` (`sort_by_id`)
4. **Reanchor** prev pointers to form a single valid chain (`reanchor`)

Signatures are cleared to `None` during reanchor; well-formedness is preserved because all content-based axioms (scope, confidence, actor, event type) are preserved by `event_content_eq`.

## Algebraic Structures

### Join-Semilattice (Status)

| Element | Coq Name | Mathematical Role |
|---------|----------|-------------------|
| Carrier | `status` | `{PROVISIONAL, FORKED, VALIDATED, DEPRECATED, ARCHIVED}` |
| Order | `status_leq` | Rank comparison `status_rank s1 <= status_rank s2` |
| Join | `status_max` | `if rank s1 <= rank s2 then s2 else s1` |
| Bottom | `PROVISIONAL` | Identity for `status_max` |
| LUB | `status_lub` | `fold_left status_max` over a list |

**Lemmas proved:** reflexivity, transitivity, upper bound, least upper bound, commutativity, monotonicity (left and right), distributivity over `++`.

### G-Counter (Confidence)

| Element | Coq Name | Mathematical Role |
|---------|----------|-------------------|
| Carrier | `nat` | Natural numbers (scaled by 100) |
| Join | `Nat.max` | Maximum over event confidences |
| Bottom | `0` | Zero confidence |
| Derived | `derived_confidence_events` | Filter-then-max over `VALIDATE`/`SNAPSHOT` |

**Lemmas proved:** monotonicity w.r.t. append, boundedness by `max_confidence`, `γ_agg` boundedness by `MAX_SCALED`.

## Historical Note: From Stubs to Full Definitions

- **Initial skeleton (pre-2025-07-03):** 6 of the 12 axioms were stubbed as `True`:
  - Axiom 5 (`axiom_precondition_eval`) — placeholder
  - Axiom 7 (`axiom_shacl_constraints`) — placeholder
  - Axiom 8 (`axiom_status_transition`) — placeholder
  - Axiom 10 (`axiom_lifecycle_monotonic`) — placeholder
  - Axiom 11 (`axiom_validator_collusion`) — placeholder
  - Axiom 12 (`axiom_synthetic_tagging`) — placeholder

- **Current (2025-07-03):** All 12 axioms are **fully defined** with closed Coq definitions. The T7 proof (`well_formedness_preservation`) was expanded from ~280 lines to **490 lines** to prove preservation for every axiom individually (via `axiom_precondition_eval_append`, `axiom_lifecycle_monotonic_append`, `axiom_validator_collusion_append`, `axiom_synthetic_tagging_append`, etc.).

```bash
cd formal/coq

```

To run `coqchk` (kernel verification):

```bash
cd formal/coq
make validate
```

This invokes `coqchk -silent -o` on all compiled `.vo` files, confirming that every proof is kernel-checked and that no axioms are used outside `Crypto.v`.
