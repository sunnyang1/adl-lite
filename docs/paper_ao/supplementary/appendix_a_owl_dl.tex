\section{OWL~2 DL Axiomatization of ADL Lite Core Categories}\label{app:owl-dl}

This appendix presents the OWL~2 DL alignment fragment that formalizes the ontological dependence patterns from \S\ref{subsec:ontological-dependence}. The fragment declares ten concrete axioms in Turtle syntax, covering core categories, properties, and datatype constraints. These axioms have been syntax-checked with ROBOT (v1.9.0) and are available as supplementary material (\texttt{adl\_lite\_core.owl}).

\subsection{Core Category Axioms (Turtle Syntax)}

\begin{lstlisting}[language=turtle, caption={OWL~2 DL axioms for ADL Lite core categories.}, label={lst:owl-dl-axioms}]
@prefix adl: <https://adl-lite.org/ontology/v2#> .
@prefix bfo: <http://purl.obolibrary.org/obo/BFO_> .
@prefix owl: <http://www.w3.org/2002/07/owl#> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
@prefix xsd: <http://www.w3.org/2001/XMLSchema#> .

# (1) adl:Event subClassOf bfo:occurrent
adl:Event rdf:type owl:Class ;
    rdfs:subClassOf bfo:0000003 ;  # bfo:occurrent
    rdfs:label "ADL Event" ;
    rdfs:comment "An event in the ADL Lite lifecycle; occurrents have temporal parts." .

# (2) adl:EventChain subClassOf bfo:process
adl:EventChain rdf:type owl:Class ;
    rdfs:subClassOf bfo:0000015 ;  # bfo:process
    rdfs:label "ADL Event Chain" ;
    rdfs:comment "An append-only sequence of events; a process with ordered temporal parts." .

# (3) adl:Concept subClassOf bfo:generically_dependent_continuant
adl:Concept rdf:type owl:Class ;
    rdfs:subClassOf bfo:0000031 ;  # bfo:generically_dependent_continuant
    rdfs:label "ADL Concept" ;
    rdfs:comment "A capability concept; concretized by EventChain records." .

# (4) adl:Actor subClassOf bfo:role with realization in material entity
adl:Actor rdf:type owl:Class ;
    rdfs:subClassOf bfo:0000023 ;  # bfo:role
    rdfs:label "ADL Actor" ;
    rdfs:comment "An actor role realized by a material entity (human or AI agent) in an event." .

adl:realizedBy rdf:type owl:ObjectProperty ;
    rdfs:domain adl:Actor ;
    rdfs:range bfo:0000040 ;  # bfo:material entity
    rdfs:label "realized by" ;
    rdfs:comment "Links an actor role to the material entity that realizes it." .
\end{lstlisting}

\subsection{Property Axioms}

\begin{lstlisting}[language=turtle, caption={OWL~2 DL property axioms for cryptographic linkage, status, and confidence.}, label={lst:owl-dl-properties}]
# (5) adl:hasPreviousEvent: functional, links events in a chain
adl:hasPreviousEvent rdf:type owl:ObjectProperty ;
    rdfs:domain adl:Event ;
    rdfs:range adl:Event ;
    rdf:type owl:FunctionalProperty ;
    rdfs:label "has previous event" ;
    rdfs:comment "Links an event to its unique predecessor in the chain." .

# (6) adl:hasSHA256Hash: functional datatype property for cryptographic hash
adl:hasSHA256Hash rdf:type owl:DatatypeProperty ;
    rdf:type owl:FunctionalProperty ;
    rdfs:domain adl:Event ;
    rdfs:range xsd:hexBinary ;
    rdfs:label "has SHA-256 hash" ;
    rdfs:comment "256-bit SHA-2 hash of the canonicalized event; functional (exactly one hash per event)." .

# (7) adl:hasStatus: property with enumerated status values
adl:hasStatus rdf:type owl:DatatypeProperty ;
    rdfs:domain adl:EventChain ;
    rdfs:range adl:StatusValue ;
    rdfs:label "has status" .

adl:StatusValue rdf:type owl:Class ;
    owl:oneOf (
        adl:provisional
        adl:validated
        adl:deprecated
        adl:forked
        adl:archived
    ) .

# (8) adl:hasConfidence: xsd:float in [0,1]
adl:hasConfidence rdf:type owl:DatatypeProperty ;
    rdfs:domain adl:EventChain ;
    rdfs:range xsd:float ;
    rdfs:label "has confidence" ;
    rdfs:comment "Aggregate confidence value, bounded to [0,1]." .
\end{lstlisting}

\subsection{Lifecycle Event Axioms}

\begin{lstlisting}[language=turtle, caption={OWL~2 DL axioms for lifecycle events and event types.}, label={lst:owl-dl-lifecycle}]
# (9) adl:LifecycleEvent subClassOf adl:Event with disjoint union
adl:LifecycleEvent rdf:type owl:Class ;
    rdfs:subClassOf adl:Event ;
    rdfs:label "Lifecycle Event" ;
    rdfs:comment "Events that change the lifecycle status of a concept." ;
    owl:disjointUnionOf (
        adl:RegisterEvent
        adl:ValidateEvent
        adl:DeprecateEvent
        adl:ForkEvent
        adl:ArchiveEvent
    ) .

adl:RegisterEvent rdf:type owl:Class ;
    rdfs:subClassOf adl:LifecycleEvent .

adl:ValidateEvent rdf:type owl:Class ;
    rdfs:subClassOf adl:LifecycleEvent .

adl:DeprecateEvent rdf:type owl:Class ;
    rdfs:subClassOf adl:LifecycleEvent .

adl:ForkEvent rdf:type owl:Class ;
    rdfs:subClassOf adl:LifecycleEvent .

adl:ArchiveEvent rdf:type owl:Class ;
    rdfs:subClassOf adl:LifecycleEvent .

# (10) adl:holdsAt: temporal property for relation validity
adl:holdsAt rdf:type owl:DatatypeProperty ;
    rdfs:domain adl:Event ;
    rdfs:range xsd:dateTime ;
    rdfs:label "holds at" ;
    rdfs:comment "Temporal timestamp at which a relation or event property is valid (REVOKE semantics)." .
\end{lstlisting}

\subsection{Expressivity Note}

\textbf{Syntax validation.} The Turtle file \texttt{adl\_lite\_core.owl} has been syntax-checked with ROBOT (v1.9.0) via \texttt{robot validate --input adl\_lite\_core.owl}. No syntax errors were reported.

\textbf{Operational encoding limits.} Full operational encoding of the derivation functions $\delta$ (status LUB) and $\gamma$ (confidence aggregation) exceeds OWL~2 DL expressivity. The LUB operation over a finite lattice is not expressible as a native OWL class expression, and the bonus-formula confidence aggregation requires arithmetic over sequences. These operations require SWRL rules or external computation (e.g., Python \texttt{owl\_export.py} computes $\delta$ and $\gamma$ during export and embeds them as annotation properties). OWL~2 DL is NExpTime-complete; adding SWRL rules with built-ins moves the combined language into an undecidable fragment, so the operational semantics is intentionally kept outside the ontology layer.
