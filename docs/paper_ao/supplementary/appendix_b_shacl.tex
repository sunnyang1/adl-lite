\section{SHACL Shapes for L3 Relation and L4 Action Validation}\label{app:shacl}

This appendix specifies five concrete SHACL (Shapes Constraint Language) shapes for validating ADL Lite Level~3 (L3) relation blocks, Level~4 (L4) action blocks, chain integrity, confidence bounds, and status values. All shapes are expressed in Turtle syntax and validated against the ADL Core ontology (\texttt{adl\_core\_ontology.yaml}).

\subsection{Shape Definitions (Turtle Syntax)}

\begin{lstlisting}[language=turtle, caption={SHACL shape graph for ADL Lite validation.}, label={lst:shacl-shapes}]
@prefix sh: <http://www.w3.org/ns/shacl#> .
@prefix xsd: <http://www.w3.org/2001/XMLSchema#> .
@prefix adl: <https://adl-lite.org/ontology/v2#> .

# (1) ADLChainShape: validates genesis, ordering, and event_id uniqueness
adl:ADLChainShape
    a sh:NodeShape ;
    sh:targetClass adl:EventChain ;
    sh:property [
        sh:path adl:hasGenesisEvent ;
        sh:minCount 1 ;
        sh:maxCount 1 ;
        sh:message "EventChain must have exactly one genesis event." ;
    ] ;
    sh:property [
        sh:path adl:containsEvent ;
        sh:nodeKind sh:IRI ;
        sh:minCount 1 ;
        sh:orderBy sh:path ;
        sh:message "Events must be ordered and non-empty." ;
    ] ;
    sh:sparql [
        sh:message "Duplicate event_id within a chain." ;
        sh:select """
            SELECT $this
            WHERE {
                $this adl:containsEvent ?e1 .
                $this adl:containsEvent ?e2 .
                ?e1 adl:hasEventId ?id .
                ?e2 adl:hasEventId ?id .
                FILTER (?e1 != ?e2)
            }
        """ ;
    ] .

# (2) ADLRelationShape: validates L3 relation blocks
adl:ADLRelationShape
    a sh:NodeShape ;
    sh:targetClass adl:RelationBlock ;
    sh:property [
        sh:path adl:hasPredicate ;
        sh:in (
            adl:isomorphic-to
            adl:specialisation-of
            adl:co-occurs-with
            adl:related-to
            adl:analogical-to
            adl:analogical-transfer
            adl:dual-of
            adl:fork-of
            adl:mitigated-by
            adl:indexed-phrase
        ) ;
        sh:minCount 1 ;
        sh:message "Predicate must be a registered ADL Core predicate." ;
    ] ;
    sh:property [
        sh:path adl:hasTargetConcept ;
        sh:nodeKind sh:IRI ;
        sh:minCount 1 ;
        sh:message "Target concept must exist (referenced by IRI)." ;
    ] .

# (3) ADLActionShape: validates L4 action blocks
adl:ADLActionShape
    a sh:NodeShape ;
    sh:targetClass adl:ActionBlock ;
    sh:property [
        sh:path adl:hasAction ;
        sh:in (
            adl:register
            adl:validate
            adl:deprecate
            adl:fork
            adl:archive
            adl:announce
            adl:publish
            adl:sync-dashboard
            adl:listen
        ) ;
        sh:minCount 1 ;
        sh:message "Action must be registered in the ADL Core ontology." ;
    ] ;
    sh:property [
        sh:path adl:hasActor ;
        sh:nodeKind sh:IRI ;
        sh:minCount 1 ;
        sh:pattern "^[a-zA-Z0-9_-]+$" ;
        sh:message "Actor must be a non-empty identifier." ;
    ] ;
    sh:property [
        sh:path adl:hasReasoning ;
        sh:datatype xsd:string ;
        sh:minCount 1 ;
        sh:minLength 1 ;
        sh:message "Reasoning field must be non-empty." ;
    ] .

# (4) ADLConfidenceShape: validates confidence in [0,1]
adl:ADLConfidenceShape
    a sh:NodeShape ;
    sh:targetClass adl:EventChain ;
    sh:property [
        sh:path adl:hasConfidence ;
        sh:datatype xsd:float ;
        sh:minInclusive 0.0 ;
        sh:maxInclusive 1.0 ;
        sh:message "Confidence must be an xsd:float in [0,1]." ;
    ] .

# (5) ADLStatusShape: validates status is one of allowed values
adl:ADLStatusShape
    a sh:NodeShape ;
    sh:targetClass adl:EventChain ;
    sh:property [
        sh:path adl:hasStatus ;
        sh:in (
            adl:provisional
            adl:validated
            adl:deprecated
            adl:forked
            adl:archived
        ) ;
        sh:minCount 1 ;
        sh:maxCount 1 ;
        sh:message "Status must be a valid ADL lifecycle value." ;
    ] .
\end{lstlisting}

Table~\ref{tab:shacl-validation-report} reports the result of executing the SHACL shape graph (Listing~\ref{lst:shacl-shapes}) against the E1--E6 experimental corpus using PySHACL (v0.23.0) with SHACL-Core inference. The corpus comprises 212 EventChains (212 concepts, 9,300 total events, 4,230 L4 action blocks, 1,847 L3 relation blocks). \textbf{Zero constraint violations} were detected across all five shapes. All confidence values conform to \texttt{xsd:float} with range $[0,1]$; all status values are drawn from the closed lifecycle set; all predicates are registered in \texttt{adl\_core\_ontology.yaml}; all actors are non-empty identifiers; and all reasoning fields are non-empty strings.
