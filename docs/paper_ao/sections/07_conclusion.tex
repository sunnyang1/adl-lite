\subsection{Summary of Contributions}\label{subsec:summary}

This paper presented ADL Lite, an event-first, Markdown-native capability-lifecycle registry. We analyzed its core categories through BFO, DOLCE, and UFO (Section~\ref{sec:ontological-analysis}); formalized a compact derivation semantics with nine proven properties including determinism, fork determinism, and confidence boundedness (Section~\ref{sec:architecture}); and empirically verified the architecture on 201 EventChains (9,300 events) with zero integrity failures and linear scalability, extended by four additional experiments (E27--E30) demonstrating $10^6$-event scalability with split-lock concurrency and incremental verification, integrity under 10{,}000-agent contention, FAISS vector index recall ${>}0.80$, and safe LLM-driven canonicalization (Section~\ref{sec:empirical-validation}). As of v0.5.0-alpha, the system also incorporates Merkle transparency anchors for $O(\log n)$ batch verification and zstd+msgpack cold storage compression. ADL Lite is not a competing ontology production method; it is a complementary governance layer that records, validates, and governs the lifecycle of capabilities produced by any agent pipeline.\par

\subsection{Future Work}\label{subsec:future-work}

We organize future work into four priority tiers, each addressing a limitation from Section~\ref{subsec:limitations}.

\textbf{Tier 1 (Critical):} \textbf{FW1}---automated OWL~2 DL alignment with BFO, DOLCE, and UFO using LogMap/AML, followed by ROBOT consistency validation. \textbf{FW4}---full W3C DID and Linked Data Proof integration, replacing self-declared string identifiers with authenticated DIDs (Phase 2). \textbf{FW9}---Ontological Assertion Market: staking-based validation with slashing for overconfidence or collusion.

\textbf{Tier 2 (High):} \textbf{FW5}---domain evaluation with IRB-approved human expert ratings on the IBM AML dataset, moving beyond synthetic pattern injection and the simulated proxy (E32). \textbf{FW6}---SHACL-governed L2 Markdown validation for machine-checkable document structure independent of the Python runtime. \textbf{FW10}---complete machine-checked proofs of Theorems 1--9 in Coq (Iris separation logic); partial progress is already made with TLA$^+$ bounded model checking (Theorems 1--3, 7, 9) and Coq proofs (Theorems 3, 4, 7, 9 fully closed). A randomized trace checker (E25, 10,000 traces, 100\% pass rate) provides interim validation. \textbf{FW12}---mechanized verification of the two-level ontological axioms and precondition decidability.

\textbf{Tier 3 (Medium):} \textbf{FW7}---extend LLM-driven canonicalization (validated in E30 with dry-run) to production mode with real LLM backends and human-in-the-loop review, plus RAG-injected domain knowledge for improved concept discovery. \textbf{FW8}---SHACL and JSON-LD export pathways for standards-compliant interoperability.

\textbf{Tier 4 (Long-term):} \textbf{FW13}---integration with emerging agent registry standards (MCP, A2A, AGNTCY, NANDA, Entra) through adapter modules.

The three-phase authentication roadmap (Phase 1~current, Phase 1.5~implemented, Phase 2~planned, Phase 3~future) is detailed in Appendix~K, together with the hash-stability migration path and the impact on $\delta$, $\gamma$, and preconditions.
