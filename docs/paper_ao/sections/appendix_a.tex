\textbf{Information Artifact Ontology (IAO) alignment.} The IAO \cite{iao} provides a rigorous framework for information content entities (ICEs) that complements PROV-O's activity-centric model. In IAO terms, an ADL Lite Concept is an ICE that is \emph{about} the domain entities it describes (e.g., a money-laundering pattern), whereas the EventChain record is the \emph{information bearer} that concretizes the ICE. The SHA-256 hash of the genesis event serves as the \emph{identifier} (IAO:\texttt{identifier}) for the Concept, whereas the \texttt{concept\_id} is a human-readable \emph{symbol} (IAO:\texttt{symbol}). This two-level identification scheme---content-addressed hash as primary key, human-readable label as alias---aligns with IAO's distinction between \emph{denotations} (what the identifier refers to) and \emph{concretizations} (how the identifier is physically realized).

\textbf{Round-trip fidelity.} The PROV-O/IAO export preserves all event fields, cryptographic hashes, and linkage structure. Re-importing the Turtle file into ADL Lite reconstructs the original EventChain with identical \texttt{event\_id}s and hashes, confirming lossless round-trip translation. The translation does not preserve: (i)~the deterministic derivation functions ($\delta$, $\gamma$) are external to PROV-O and must be supplied by the consumer; (ii)~the precondition rules are not expressible in PROV-O's descriptive model and require ADL Lite's ActionExecutor. Thus, the PROV-O export captures \emph{provenance description} but not \emph{governance enforcement}.
