LLM agents proliferate across ecosystems, each exposing tools, APIs, and knowledge-domain capabilities, yet no lightweight, tamper-evident, lifecycle-aware registry exists to govern what a capability claims, how it evolves, or whether it remains trustworthy. ADL Lite addresses this gap as an event-first, Markdown-native capability-lifecycle registry. Capabilities are modeled as append-only, cryptographically linked EventChains, with all lifecycle state (status, confidence, validators) derived exclusively from event history via deterministic functions. We present a two-level ontological analysis aligning ADL Lite with BFO, DOLCE, and UFO; a compact formal specification with nine proven properties (Theorems 1--9); and architectural verification on 201 EventChains (9,300 events) demonstrating linear-time verification, zero integrity failures, and deterministic precondition enforcement under a collaborative non-Byzantine trust model.
