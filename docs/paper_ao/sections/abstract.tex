LLM agents proliferate across ecosystems, each exposing tools, APIs, and knowledge-domain capabilities, yet no lightweight, tamper-evident, lifecycle-aware registry exists to govern what a capability claims, how it evolves, or whether it remains trustworthy. ADL Lite addresses this gap as an event-first, Markdown-native capability-lifecycle registry. Capabilities are modeled as append-only, cryptographically linked EventChains, with all lifecycle state (status, confidence, validators) derived exclusively from event history via deterministic functions. We present a two-level ontological analysis aligning ADL Lite with BFO, DOLCE, and UFO; a compact formal specification with nine proven properties (Theorems 1--9); v0.5.0-alpha architectural enhancements including split-lock concurrency for $10^4$-agent contention, FAISS-backed vector indexing for semantic search, Merkle transparency anchors for $O(\log n)$ batch verification, and LLM-driven canonicalization for near-duplicate concept merging; and empirical validation on 201 EventChains (9,300 events) plus four new experiments demonstrating linear-time verification at $10^6$ events, zero integrity failures under 10K concurrent agents, ${>}0.80$ vector index recall, and safe LLM-driven normalization.
