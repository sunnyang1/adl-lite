# 01 — 故障定级框架

> 版本：v1.0 | 最后更新：2026-06-04

---

## 故障级别定义

| 级别 | 名称 | 判定标准 | 响应时间 | 通报频率 | 升级对象 |
|------|------|---------|---------|---------|---------|
| **SEV1** | 致命 | 全服务不可用、数据丢失风险、安全漏洞 | < 5 分钟 | 每 15 分钟 | VP Eng + CTO 立即通知 |
| **SEV2** | 严重 | >25% 用户受影响、核心功能不可用 | < 15 分钟 | 每 30 分钟 | 技术经理 15 分钟内 |
| **SEV3** | 一般 | 非核心功能异常、有临时绕过方案 | < 1 小时 | 每 2 小时 | 团队 Leader 下次站会 |
| **SEV4** | 轻微 | 外观问题、无用户影响、技术债触发 | 下一工作日 | 每日 | 纳入 backlog 排序 |

---

## 详细判定标准

### SEV1 — 致命故障

满足以下**任一条件**即为 SEV1：

- [ ] **全服务宕机**：核心业务入口无法访问，所有用户均受影响
- [ ] **数据安全**：数据泄露、越权访问、SQL 注入等安全漏洞被利用
- [ ] **数据丢失/损坏**：用户数据丢失或不可逆损坏
- [ ] **资金风险**：支付/交易链路异常，涉及资金错误
- [ ] **合规风险**：违反法律法规，可能导致处罚
- [ ] **SLO 严重违反**：错误预算在 1 小时内耗尽

**SEV1 动作清单：**
- [ ] 立即在 #incidents 频道声明，@vp-eng @cto
- [ ] 5 分钟内 IC 到位，启动故障指挥
- [ ] 每 15 分钟发送状态更新
- [ ] 30 分钟内未定位根因 → 自动升级，增加人力
- [ ] 通知客服/运营团队准备用户沟通

### SEV2 — 严重故障

满足以下**任一条件**：

- [ ] **大面积影响**：>25% 用户无法使用核心功能
- [ ] **核心功能不可用**：登录、支付、数据查询等核心链路中断
- [ ] **性能严重劣化**：P99 延迟超过 SLO 阈值 10 倍
- [ ] **付费用户影响**：任一付费客户报障且确认为系统问题
- [ ] **外部依赖不可用**：关键第三方服务中断且无降级

**SEV2 动作清单：**
- [ ] 在 #incidents 频道声明，@tech-leads
- [ ] 15 分钟内 IC 到位
- [ ] 每 30 分钟发送状态更新
- [ ] 2 小时内未定位根因 → 升级为 SEV1 响应级别
- [ ] 更新内部状态页

### SEV3 — 一般故障

- [ ] **非核心功能异常**：辅助功能不可用，主流程不受影响
- [ ] **小范围影响**：<25% 用户受影响
- [ ] **有可行的绕过方案**：用户可通过其他方式完成操作
- [ ] **定时任务失败**：非实时性批处理任务异常

**SEV3 动作清单：**
- [ ] 在 #incidents 频道声明
- [ ] 1 小时内确认问题
- [ ] 每 2 小时更新进展
- [ ] 下一工作日完成初步分析

### SEV4 — 轻微故障

- [ ] **外观问题**：UI 错位、文案错误等不影响功能
- [ ] **无用户影响**：内部工具问题、开发环境异常
- [ ] **技术债触发**：已知非紧急问题再次出现
- [ ] **非生产环境**：预发/测试环境问题

**SEV4 动作清单：**
- [ ] 创建工单或在 #incidents 频道记录
- [ ] 下一工作日排期处理
- [ ] 不需要实时通报

## 自动升级规则

以下情况**自动触发**故障级别升级：

| 触发条件 | 升级动作 |
|---------|---------|
| 影响范围翻倍 | 升一级 |
| SEV1 30 分钟未定位根因 | 通知 VP Eng，增加人力 |
| SEV2 2 小时未定位根因 | 按 SEV1 级别响应 |
| 付费客户报障 | 最低 SEV2 |
| 任何数据完整性疑虑 | 立即 SEV1 |
| 同一服务 7 天内重复故障 | 升一级处理 |
| 安全事件确认 | 立即 SEV1 |

## 故障声明模板

在 #incidents 频道中使用以下格式声明故障：

```
🚨 故障声明

级别：SEV[1-4]
服务：[受影响服务名称]
影响：[影响描述，包括用户比例]
IC：[故障指挥官姓名]
TL：[技术负责人]
CL：[沟通负责人]

开始时间：[故障开始时间]
下次更新：[预计下次更新时间]

简要描述：[一句话描述发生了什么]
```

## 故障状态流转

```
[正常运行] → [调查中] → [已定位] → [修复中] → [监控中] → [已恢复]
                  ↓           ↓          ↓          ↓
              [升级]      [升级]     [升级]     [回滚]
```

每次状态变更时，由 CL 在 #incidents 频道更新状态标签。
