\begin{abstract}
The proliferation of large language model (LLM) based autonomous agents has intensified interest in multi-agent systems capable of collaborative knowledge synthesis, yet the conceptual outputs generated by such systems lack typed relations, lifecycle traceability, and machine-checkable provenance. This paper presents ADL~Lite, an event-first operational ontology in which concepts are modeled as append-only, cryptographically hashed EventChains and all lifecycle state---status, confidence, validators---is derived exclusively from event history. Formal derivation semantics specify status computation and confidence aggregation as deterministic functions over the event sequence, with six proved properties including determinism (Theorem~1), confluence under fork (Theorem~2), confidence boundedness (Theorem~4), and Sybil-resistant aggregation (Corollary~2). A precise canonical serialization with recursive key sorting and six-decimal float normalization ensures platform-independent SHA-256 hashing; timestamps are deliberately excluded from hash input to tolerate arbitrary clock skew across distributed agents. nine experiments establishing chain integrity (100\% P/R on 60 chains), exhaustive status derivation (2,204/2,204 correct), snapshot round-trip (38/38 match), precondition enforcement (P=1.0, R=1.0, F1=1.0 across 13 test cases), multi-agent auditability (5/5 chains), real-time pattern detection (6/6 tests, 0 false positives), edge offline synchronization (5/5 tests, zero merge conflicts), Git baseline comparison (per-event diagnostic precision and semantic violation classification), and scale validation (495,671 chains / 5,080,714 events in 238~s, zero failures). Appendices provide complete, syntactically valid PROV-O export (Appendix~A) and SHACL shape validation (Appendix~B) examples. ADL~Lite is pip-installable, operates over Markdown files in standard Git repositories, requires no triple store or enterprise infrastructure, and achieves deployment-ready lifecycle traceability under a collaborative-audit threat model with explicit paths to Linked Data Proofs and CRDT-based merging.
\keywords{ontology engineering; event sourcing; multi-agent systems; knowledge graphs; anti-money laundering; provenance; PROV-O}
\end{abstract}
